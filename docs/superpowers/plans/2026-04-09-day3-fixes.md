3. In `add_transformed_position_to_tracks()`, clamp `position_transformed` to [0,105]×[0,68]

```python
def clamp_pitch_position(x: float, y: float,
                          x_max: float = 105.0, y_max: float = 68.0):
    """Clamp a transformed pitch coordinate to valid field bounds."""
    return (max(0.0, min(float(x), x_max)),
            max(0.0, min(float(y), y_max)))


def _linear_fill_keypoints(sampled: dict, total_frames: int) -> list:
    """
    Fill un-sampled frames by linearly interpolating each keypoint ID
    between known frames. Nearest-neighbor fallback for edge gaps.

    Args:
        sampled: {frame_idx: {keypoint_id: [x, y]}} — only sampled frames
        total_frames: total number of frames in the video

    Returns:
        list of length total_frames, each element is {keypoint_id: [x, y]}
    """
    if not sampled:
        return [{} for _ in range(total_frames)]

    # Collect all keypoint IDs seen across all sampled frames
    all_kids = set()
    for kps in sampled.values():
        all_kids.update(kps.keys())

    # Build per-keypoint time series and interpolate with pandas
    import pandas as pd
    result = [{} for _ in range(total_frames)]

    # Copy known frames into result first
    for fi, kps in sampled.items():
        if fi < total_frames:
            result[fi] = dict(kps)

    for kid in all_kids:
        # Build series of known (frame_idx, x) and (frame_idx, y)
        known_idx = [fi for fi, kps in sampled.items() if kid in kps and fi < total_frames]
        if len(known_idx) < 2:
            # Only one known point — copy to all missing frames (nearest-neighbor)
            if known_idx:
                fi0 = known_idx[0]
                val = sampled[fi0][kid]
                for fi in range(total_frames):
                    if kid not in result[fi]:
                        result[fi][kid] = list(val)
            continue

        xs = pd.Series({fi: sampled[fi][kid][0] for fi in known_idx},
                       dtype=float).reindex(range(total_frames))
        ys = pd.Series({fi: sampled[fi][kid][1] for fi in known_idx},
                       dtype=float).reindex(range(total_frames))

        xs = xs.interpolate(method="linear").bfill().ffill()
        ys = ys.interpolate(method="linear").bfill().ffill()

        for fi in range(total_frames):
            if kid not in result[fi] and not pd.isna(xs[fi]):
                result[fi][kid] = [float(xs[fi]), float(ys[fi])]

    return result
```

### Context for the implementer

After the streaming refactor, `run_global_analysis()` currently reads the video file **3 separate times**:
1. Step 2: `tracker.get_object_tracks_streamed(video_path, total)` — streams the file once
2. Step 3: `cam.get_camera_movement_streamed(video_path, total)` — streams the file again
3. Step 4: `kp.predict_streamed(video_path, total)` — streams the file a third time

For short videos (≤3000 frames, ~2 min at 24fps), reading once into memory is faster because:
- Disk I/O is amortized across all 3 uses
- The non-streaming methods (`get_object_tracks`, `get_camera_movement`, `predict`) run the same logic but share the same frames list

**The non-streaming versions already exist** in `analysis_core.py`:
- `tracker.get_object_tracks(frames)` — takes frames list
- `cam.get_camera_movement(frames)` — takes frames list
- `kp.predict(frames)` — takes frames list

`read_video` was removed from the imports in `tasks.py` during the streaming refactor. We need to add it back.

**For videos >3000 frames**: keep the streaming path (memory safety for 20-90 min games).
